# Token 消耗模型说明

MiMo Code Surgeon 的 token 需求来自真实代码仓库维护任务，而不是简单文本生成。

## 主要消耗来源

1. 仓库扫描与长上下文理解  
   读取 README、源码、测试、依赖文件、配置文件、Issue、CI 日志。

2. 多 Agent 上下文传递  
   架构 Agent、检索 Agent、实现 Agent、测试 Agent、Review Agent、文档 Agent 之间需要传递中间产物。

3. 多轮代码生成与修复  
   初始 patch 可能失败，需要读取测试日志、定位原因、二次修复。

4. 回归验证与文档生成  
   需要生成 PR 描述、迁移说明、风险评估、测试摘要和运行报告。

## 预计消耗

- 单个中大型仓库初始化：50 万至 200 万 tokens。
- 单个真实 Issue 完整处理：100 万至 500 万 tokens。
- 每天处理 20 至 50 个真实 Issue：约 3,000 万至 2 亿 tokens。

## 为什么需要较高额度

真实工程任务通常不是一次 prompt 就能完成，而是需要：

```text
长上下文读取 -> 计划生成 -> 补丁生成 -> 测试失败 -> 日志分析 -> 二次修复 -> 回归验证 -> 文档生成
```

每一步都需要模型理解前序结果，因此需要稳定的大额度 token 支持。
